# Hireme - Sistema de Recrutamento

Solucao para o desafio tecnico de Desenvolvedor Junior: otimizar o processo de recrutamento do setor de RH, centralizando dados de candidatos, automatizando notificacoes e integrando com ferramentas corporativas.

---

## Contexto

A Maria, Head de RH, precisa de uma solucao que:
- Disponibilize um **unico link** para candidatos preencherem seus dados
- **Notifique** o time de RH por **e-mail** e **Microsoft Teams** a cada envio
- **Armazene** os dados automaticamente em **duas pastas distintas no SharePoint**

---

## Requisitos Obrigatorios

| # | Requisito | Descricao |
|---|-----------|-----------|
| 1 | Formulario de candidatura | Interface web com link unico para candidatos preencherem seus dados |
| 2 | Notificacao por e-mail | Enviar e-mail ao time de RH quando um candidato submeter o formulario |
| 3 | Notificacao via Microsoft Teams | Enviar mensagem em canal/chat do Teams a cada nova candidatura |
| 4 | Armazenamento no SharePoint | Salvar os dados automaticamente em duas pastas distintas no SharePoint |
| 5 | Automacao de fluxo | Orquestrar todo o pipeline (formulario -> notificacoes -> armazenamento) de forma automatica |

---

## Diferenciais

| # | Diferencial | Descricao |
|---|-------------|-----------|
| 1 | Validacao de dados | Validar campos do formulario (e-mail, telefone, campos obrigatorios, formatos) |
| 2 | Dashboard de visualizacao | Painel para o RH acompanhar candidaturas, metricas e status |
| 3 | Classificacao automatica de candidatos | Categorizar/pontuar candidatos com base em criterios definidos |
| 4 | Integracao com banco de dados | Persistir dados em banco relacional ou NoSQL alem do SharePoint |
| 5 | Uso de APIs | Expor/consumir APIs REST para comunicacao entre servicos |
| 6 | Logs ou monitoramento | Registrar eventos, erros e metricas da aplicacao |
| 7 | Interface customizada | Frontend proprio e estilizado (nao depender de formularios genericos) |
